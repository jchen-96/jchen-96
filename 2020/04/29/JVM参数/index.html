JVM之JVM参数

发表于 2020-04-29 | 分类于 Java ， JVM

JVM参数

1、JVM参数总结

-XX: 这是固定开头的位置

  • 布尔类型的参数

-XX:+

-XX:-

  • 一般类型的参数

-XX:

2、用到的

  • -XX:+TraceClassLoading 用于追踪类的加载信息并打印出来。
  • -XX:+TraceClassUnloading 可以用于追踪类的卸载信息并打印出来。
  • -XX:+HeapDumpOnOutOfMemoryError 将当时的现场信息转储下来

3、其他参数

-Xms5m -Xmx5m 设置堆空间的大小，-Xms5m最小值5M，-Xmx5m 最大值5m

-Xss160k 设置虚拟机栈的大小

  • 限制元空间的大小，以及阻止元空间的自动扩容
  • -XX:MaxMetaspaceSize=10m

助记符

助记符

  • ldc 表示将int，float或者String类型的常量值从常量池推送至栈顶。
  • bipush表示将单字节的常量值(-128-127)从常量池推送至栈顶
  • sipush表示将一个短整型(-32768-32767)常量值从常量池推送至栈顶
  • iconst_1 表示将一个int类型的1从常量池推送至栈顶。
  • iconst_2,iconst_3,iconst_4,iconst_5 iconst_0,iconst_m1(-1)这七个有助记符
  • anewarray 表示创建一个引用类型的(如类，接口，数组)数组，并将其值压入栈顶。
  • newarray 表示创建一个指定的原始类型(int double float char 等)的数组，并将其引用值压入栈顶。

命令

  • javap -c 展示字节码文件
  • javap -verbose 更加详细的展示字节码文件。
  • javap -verbose -p 展示所有的字节码文件，包括私有属性
  • java -Djava.ext.dirs=./ MyClass 修改运行时候的java扩展加载器的目录，注意对于扩展类加载器，需要将文件打包成jar包才可以被拓展类加载器扫描到。
  • System.getProperty(“sun.boot.class.path”) 启动类加载器的路径
  • System.getProperty(“java.ext.dirs”) 拓展类加载器的路径
  • System.getProperty(“java.class.path”) 应用类类加载器的路径、
  • System.getProperties().put(“sun.misc.ProxyGenerator.saveGeneratedFiles”,”true”); 通过修改系统属性，可以将动态代理过程中生成的文件写到磁盘中
