链表基础

发表于 2017-03-26 | 分类于 Algorithm ， LeetCode

链表问题一般都是要求经过调整指针的next节点的值进行相关的调整 持续更新中。。。。。。。。。

经典问题之反转链表(206)

  • 就是将一个链表通过next域的调整达到反转的目的。
  • 可以设置三个指针，pre,cur,next,分别指向要处理的指针的前一个指针，要处理的指针，要处理的指针的后一个指针。
  • 小坑：开始如果将head设为pre指针会造成，对于两个链表的链表无法正确反转，如果将head设为cur,在处理最后一个节点的时候，会导致next指针出现空引用的问题。
  • 所以将next指针的初始化放在while循环中。就可以避免这种问题。
  • 虽然代码看起来确实简洁，不想清楚临界条件就会日了狗了。。。

经典问题之删除节点(203)

  • 给定一个链表的头结点，和一个指定的值，删除该链表中所有值为为该值的节点，并返回新链表的头结点
  • 通过设置虚拟头结点可以使得删除头结点的过程更加一般化。
  • 链表还是要十分注意空引用的问题，动手画一下更清楚。
  • 如果是cpp的话要注意最后释放虚拟节点的内存空间，防止内存泄露，以及循环过程中保存要删除的节点的值，循环过程中释放掉该节点的值。

链表问题之删除重复的元素(82)

  • 如原链表为 1->1->2->3->4->4->5->6。经过删除操作后返回2->3->5->6.
  • 注意空引用的问题，以及如何删除所有的重复元素，在这里我采用保存重复元素的值。然后按照(203)中的方法删除节点。
