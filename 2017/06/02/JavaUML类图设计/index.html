JavaUML类图基本概念

发表于 2017-06-02 | 分类于 Java ， 设计模式

什么是类图

类图是软件工程的同意建模语言一种静态结构图,该图秒数了系统的类集合,类的属性和类之间的关系.帮助人们简化对系统的理解,它是系统分析和设计阶段的重要产物,也是系统编码和测试的重要模型依据.

大致分类

类的UML

在UML类图中，类使用包含类名，属性，方法名及其参数并且用分割线分隔的长方形表示。

类之间的关系

  • 泛华关系(继承关系)

泛化关系可以用带空心三角形的直线来表示

  • 实现关系(接口实现)

实现关系用带空心三角形的虚线来表示

  • 依赖关系

依赖关系是类和类之间最弱的一种关系,依赖关系可以理解为一个类使用了另外一个类.
在UML中用带箭头的虚线表示依赖关系

  • 关联关系

关联关系表示一个类和另外一个类有对应联系,比老师和地址
在UML中用带箭头的实线表示依赖关系

  • 聚合关系

聚合关系是表示整体和部分的关系,但是部分可以脱离整体而存在
例如一个Teachers对象有一辆汽车Car，此时Car就是Teachers的一部分，但是Car可以脱离Teachers而存在
在UML中用带空心管菱形的直线来表示

  • 组合关系

组合关系也是一种整体和部分的关系,但是部分的存活周期受到整体的影响，若整体不存在则部分也将不存在
在UML中带实心菱形直线表示
